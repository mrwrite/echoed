# EchoEd V2 Design System

## Why

EchoEd V2 has the right platform structure, but the interface still carries page-specific admin styling from earlier phases. The platform needs a unified premium visual language that presents EchoEd as an AI-native knowledge product platform rather than a school LMS or generic CRUD console.

## What Changes

- Add reusable EchoEd Experience components for cards, heroes, sections, buttons, badges, metrics, inputs, form sections, stat cards, timelines, empty states, steppers, tags, and navigation cards.
- Refresh global design tokens toward a dark luxury theme with midnight backgrounds, frosted glass, soft purple, warm gold, and azure accents.
- Modernize shared utility classes used by V2 pages without changing routes, services, APIs, data loading, authorization, learner runtime, or business behavior.
- Apply the premium visual layer to key V2 surfaces: Workspace, Product Studio, Review Center, Learner Portal, and Analytics.

## Non-Goals

- No routing changes.
- No API changes.
- No business logic changes.
- No runtime education behavior changes.
- No commercialization, payments, checkout, subscriptions, marketplace, or AI generation execution.
