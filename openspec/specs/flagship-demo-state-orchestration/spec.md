## Requirements
### Requirement: Seed canonical learner archetypes

The system SHALL seed bounded learner archetypes for the flagship pathway using real platform state.

#### Scenario: Learner archetypes are prepared

- **WHEN** the demo environment is seeded
- **THEN** at least one learner represents a strong or mastered state
- **AND** at least one learner represents a struggling state
- **AND** at least one learner represents an ambiguous or monitor state
- **AND** at least one learner represents a normal progression state

### Requirement: Stable runtime recommendation examples

The system SHALL produce stable runtime recommendation examples from seeded flagship evidence using existing runtime logic.

#### Scenario: Runtime recommendations are evaluated

- **WHEN** runtime recommendations are evaluated against the seeded flagship environment
- **THEN** the seeded environment produces stable examples for enrichment, reteach, review, monitor, and normal
- **AND** no demo-only runtime shortcut logic is required

### Requirement: Stable governance demonstration states

The system SHALL seed flagship governance examples that remain staff-readable and deterministic.

#### Scenario: Governance validators read seeded flagship state

- **WHEN** publish-readiness, safe-publish, lineage-safety, and competency-integrity validators read the seeded environment
- **THEN** they produce stable demonstration examples suitable for teacher or admin walkthroughs
- **AND** those examples are derived from existing governance behavior
