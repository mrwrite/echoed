## ADDED Requirements

### Requirement: Evolutionary technical alignment
The platform SHALL preserve the current Angular frontend architecture, governed learning delivery, institutional assessment foundation, auth and session authority, UX state system, and course, unit, and lesson hierarchy as the technical baseline for multi-pathway curriculum architecture.

#### Scenario: Curriculum architecture remains evolutionary
- **WHEN** future implementation phases are planned from this architecture
- **THEN** those phases MUST build on the current institutional runtime instead of requiring a rewrite of frontend, delivery, assessment, or session systems

### Requirement: No parallel institutional engines
The platform SHALL avoid introducing parallel curriculum systems, parallel progression systems, or fragmented assessment systems as part of the multi-pathway curriculum architecture.

#### Scenario: Architectural guardrails prevent fragmentation
- **WHEN** new pathway capabilities are proposed or implemented
- **THEN** the architecture MUST reject solutions that duplicate curriculum authority, progression authority, or assessment authority outside the canonical platform foundations
