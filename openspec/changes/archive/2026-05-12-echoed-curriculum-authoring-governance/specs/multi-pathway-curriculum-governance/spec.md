## ADDED Requirements

### Requirement: Curriculum governance scales across pathways without fragmentation
The system SHALL support flagship, academic, STEM, GED/adult, and enrichment pathways through one canonical curriculum governance system rather than parallel pathway engines.

#### Scenario: New pathway types reuse the same governance model
- **WHEN** EchoEd adds future pathways beyond the flagship implementation
- **THEN** they SHALL use the same governed curriculum lifecycle, readiness, publishing, and maintenance model

#### Scenario: Pathway metadata stays additive
- **WHEN** pathway-specific governance nuance is needed
- **THEN** the system SHALL prefer additive metadata and bounded rules instead of introducing a second curriculum system

#### Scenario: Learner progression authority remains singular
- **WHEN** curriculum governance expands across pathways
- **THEN** governed learner progression SHALL remain the canonical delivery backbone
