## ADDED Requirements

### Requirement: Canonical learning session flow
The platform SHALL define a standard EchoEd learning session structure that includes entry or activation, story or media context, guided instruction, exploration or activity, reflection or discussion, assessment or check-for-understanding, mastery signaling, remediation, enrichment, and closure.

#### Scenario: Session flow remains recognizable across lessons
- **WHEN** learners participate in lessons within the flagship pathway
- **THEN** the instructional rhythm MUST remain structurally recognizable even when content, pacing, or age-band adaptations vary

### Requirement: Session timing and visibility expectations
The platform SHALL define timing expectations, transition philosophy, learner-state expectations, and educator visibility expectations for the canonical learning session flow.

#### Scenario: Educators can interpret learner session state coherently
- **WHEN** an educator facilitates or reviews a learning session
- **THEN** the blueprint MUST define what the learner is expected to be doing and what the educator is expected to see at each major stage
