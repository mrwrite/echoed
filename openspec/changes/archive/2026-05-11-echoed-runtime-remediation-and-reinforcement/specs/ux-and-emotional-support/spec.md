## ADDED Requirements

### Requirement: Emotionally safe runtime support UX
The platform SHALL extend the premium UX foundation to support emotionally safe remediation, retry, encouragement, and continuation experiences.

#### Scenario: Failure and retry states remain supportive
- **WHEN** a learner encounters weak performance, blocked confidence, or a retry moment
- **THEN** the UX MUST use supportive state patterns, gentle transitions, and continuity-preserving messaging
- **AND** it MUST reduce shame, friction, and abrupt context loss

### Requirement: Runtime support reuses canonical UX state primitives
The platform SHALL express remediation, enrichment, retry, and encouragement states through the existing premium UX and state primitive strategy rather than inventing route-local UX systems.

#### Scenario: Support UX remains operationally coherent
- **WHEN** runtime support states are introduced
- **THEN** they MUST reuse canonical loading, blocked, retry, empty, and encouragement patterns where appropriate
- **AND** they MUST preserve current shell and route structure
