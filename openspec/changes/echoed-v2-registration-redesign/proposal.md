# EchoEd V2 Registration Redesign

## Why

Account creation should feel approachable, trustworthy, and premium. The current registration form works, but it reads like a conventional form panel and does not explain roles, organization setup, or the onboarding journey clearly enough.

## What Changes

- Redesign registration as a responsive two-column experience.
- Use the left column for account creation with clearer hierarchy, larger fields, better password controls, improved validation messaging, and progressive organization setup disclosure.
- Use the right column for benefits, role explanations, journey preview, security/trust messaging, and premium visual framing.

## Non-Goals

- No backend changes.
- No API changes.
- No route changes.
- No registration/authentication logic changes.
- No changes to form controls, validators, or submit behavior.
